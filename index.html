The problem

Every new enterprise data-sharing engagement required the same manual sequence: review the customer's data schema, cross-reference it against anonymization capabilities, evaluate privacy risk using attacker-model results, draft an assessment, and route it for compliance sign-off. Each pass took most of a week — and most of that time was re-deriving context that had already been established in a prior, similar assessment.

The trust boundary

The design constraint that shaped everything else: the customer never gets access to internal systems. They hand over documents once — everything after that runs entirely inside internal infrastructure.

How it works

01

Context gathering

The agent pulls the customer's architecture docs and data specs from an internal Confluence space — never from the customer's own systems.

02

Prior assessment search

The current use case is embedded and matched against a vector store of past assessments. A high similarity score lets the agent reuse prior analysis instead of starting cold.

03

Attacker-model decision

Without a close precedent, the pipeline triggers a full attacker-model run to quantify re-identification risk for this specific data configuration.

04

Draft generation

An LLM call produces a structured draft: threat model, quantified risk, configuration recommendation, and regulatory (GDPR) mapping.

05

Fact-check pass

A separate LLM call cross-checks every claim in the draft against the source documents and flags low-confidence sections rather than letting them pass silently.

06

Human review

The Trust/Privacy team accepts, edits, or requests a regeneration. Edits are fed back as examples to improve future drafts.

Why the fact-check step matters

The riskiest failure mode for an LLM-in-the-loop compliance workflow isn't a bad draft — it's a confident, wrong one that a reviewer rubber-stamps. Splitting generation and verification into separate LLM calls, and routing every output through a human reviewer, keeps the system a draft assistant rather than an autonomous decision-maker.
